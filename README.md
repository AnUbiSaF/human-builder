# 🏗️ HumanBuilder

[![Fabric](https://img.shields.io/badge/Minecraft-1.21.4-red.svg)](https://minecraft.net)
[![Loader](https://img.shields.io/badge/Fabric-Loader-blue.svg)](https://fabricmc.net)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

**HumanBuilder** — это Fabric-мод для Minecraft 1.21.4, предназначенный для автоматического строительства зданий из файлов схем (`.schem`, `.schematic`, `.litematic`). 

Главное отличие этого мода от обычных строительных ботов — **полная имитация поведения реального игрока (человека)**. Мод спроектирован так, чтобы его действия выглядели естественно для сторонних наблюдателей и систем защиты серверов.

---

## ✨ Основные особенности

* **🧠 Человекоподобное поведение (Human-like AI):**
  * **Плавное наведение камеры (LERP):** Никаких мгновенных разворотов головы. Камера плавно наводится на нужный блок с мягким замедлением в конце траектории.
  * **Микро-тремор рук:** Имитирует естественное дрожание прицела живого человека (дрейф амплитудой $\pm0.15^{\circ}$).
  * **Задержки и паузы:** Бот делает небольшие микро-паузы «на раздумья» между установкой блоков и длительные паузы при переходе на новые этажи здания.
  * **Имитация ошибок:** Бот иногда совершает «ошибки» установки блока, на несколько секунд останавливается, как бы осознавая её, а затем продолжает работу.
  * **Естественный бег:** Персонаж перемещается по прямой траектории и прыгает только тогда, когда физически упирается в препятствие.

* **📂 Поддержка современных форматов схем:**
  * Чтение файлов `.schem` / `.schematic` от **WorldEdit** (включая современный формат **Sponge v3**).
  * Чтение файлов `.litematic` от **Litematica**.
  * Автоматический разбор палитры блоков и NBT-тегов.

* **🛠️ Умный строительный оркестратор:**
  * **Логическая сортировка:** Сначала строится фундамент (по спирали от угла к центру), затем стены (рядами вдоль периметра), перекрытия, крыша и в последнюю очередь — декорации (двери, факелы, стёкла). Это гарантирует, что каждый блок будет иметь прочную опору в мире.
  * **Строгие слои и пакеты:** В режиме `layered` бот завершает текущий Y-слой и связанные группы одинаковых блоков перед переходом дальше, сохраняя доступ к внутренним позициям.
  * **Обязательное послойное основание:** В GUI задаётся `X` от 1 до 8. При `X=3` бот сначала полностью завершает конструктив локальных уровней `Y=1`, `Y=2` и `Y=3`; стекло и декор остаются отложенными. Только после этого продолжается архитектурная постройка выше.
  * **Временные опоры:** Если у связного фронта действительно нет постоянной опоры, бот строит короткую служебную цепочку из полноразмерного булыжника и удаляет её после завершения участка. Slab, stairs и другие блоки с формой или placement-state для опоры не используются.
  * **Корректные формы:** Лестницы устанавливаются с направлением и половиной из схемы, а плиты поддерживают состояния `top`, `bottom` и `double` с проверкой результата.
  * **Безопасный полёт:** Перед полным A* проверяется прямой свободный воздушный коридор; во время расчёта бот отпускает клавиши и не вращает камеру.
  * **Смещение к игроку:** Схема автоматически разворачивается и привязывается в 3 блоках перед лицом игрока. Больше никаких строительств на спавне `(0,0,0)`.
  * **Авто-забор из креатива:** Если в хотбаре заканчивается нужный блок, бот автоматически берёт его из креативного инвентаря в последний слот хотбара, отправляя корректные сетевые пакеты.

## 🚀 Команды управления

Все команды вводятся во внутриигровой чат:

* `H` — открыть панель управления с загрузкой схемы, прогрессом, режимом слоёв и перемещением голограммы.
* `/humanbuilder gui` — открыть ту же панель через команду.
* `/humanbuilder build <имя_файла>` — запустить строительство схемы. Файл ищется в папке `.minecraft/schematics/` (поддерживаются расширения `.schem`, `.schematic`, `.litematic`).
* `/humanbuilder stop` — прервать текущее строительство, сбросить все клавиши управления и вернуть управление игроку.
* `/humanbuilder status` — вывести в чат текущее состояние стейт-машины и процент выполнения (поставлено блоков / всего блоков).
* `/humanbuilder mode architectural` — режим по умолчанию: сначала безопасно убрать конфликты и завершить настраиваемое послойное основание, затем связно построить каркас и крышу, после них внешнее заполнение со стеклом, интерьер и финальный декор.
* `/humanbuilder mode layered` — строить строго по Y-слоям снизу вверх.

## ⚙️ Архитектура проекта

Проект разделен на логические модули:
* 📁 `parser` — разбор NBT-данных файлов схем (`SchematicParser`), декодирование палитры и битовых массивов.
* 📁 `logic` — сортировщик блоков (`BuildLogicSorter`) и классификатор категорий (`BlockCategory`), выстраивающий «человеческую» очередь установки.
* 📁 `executor` — главная стейт-машина (`BuildExecutor`), оркестрирующая состояния: `IDLE` ➔ `SORTING` ➔ `WALKING` ➔ `LOOKING` ➔ `PLACING` ➔ `WAITING`.
* 📁 `movement` — контроллер ходьбы (`MovementController`), нажимающий виртуальные WASD-клавиши и управляющий прыжками при коллизиях.
* 📁 `camera` — плавное наведение прицела (`CameraSmoother`) с LERP-фильтрацией и генератором шума тремора.
* 📁 `placer` — модуль взаимодействия с миром (`BlockPlacer`), отвечающий за выбор слота хотбара, creative-fill забор блоков и отправку пакетов клика.

## 🔨 Сборка и установка

### Требования
* **JDK 21**
* **Gradle 8.12+**

### Компиляция мода
Откройте терминал в корневой папке проекта и выполните сборку:

```bash
# Для Linux / macOS:
./gradlew build

# Для Windows (PowerShell):
$env:JAVA_HOME="C:\Program Files\Java\jdk-21"; .\gradlew build
```

Сгенерированный jar-файл мода появится по пути:
📂 `build/libs/human-builder-1.0.0.jar`

### Установка в игру
1. Установите **Fabric Loader** для Minecraft 1.21.4.
2. Поместите файл `human-builder-1.0.0.jar` в папку `.minecraft/mods/`.
3. Запустите игру.
4. Создайте в папке `.minecraft/` папку `schematics` и поместите туда файлы ваших построек (например, `magictower.schem`).
